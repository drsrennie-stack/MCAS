Step 3 · WCAG 2.2 AA checklist

Click to mark Pass. Type a remediation note to mark Fail.

Thirty operationally relevant Success Criteria, grouped by POUR principle. Empty note plus an unchecked box means Not Reviewed. Checked box means Pass. Any text in the note field flips that row to Fail and feeds the remediation report.

Step 4 · Alt text audit

Heuristic check on the alt text you've actually written.

Paste one alt-text string at a time. The tool flags common failure modes: missing alt, image-of-X redundancy, file-name leaks, length issues, and decorative-but-not-empty patterns. Catches roughly 80% of the alt mistakes I see in faculty handoffs.

Step 5 · Heading hierarchy

Paste HTML. Get the heading tree and any skipped levels.

Drop any chunk of HTML. The tool extracts the h1 through h6 outline, renders it as an indented tree, and flags any place a level was skipped (e.g., h2 jumping straight to h4). Multiple h1s also flagged. Covers WCAG 1.3.1 Info and Relationships.
